# F019.9 — Pitch actions: detail toolbar Tools menu + list delete parity

## Motivation
Two requests from Christian in one session, both about the pitch-action surfaces:

1. **Detail toolbar overflowed.** The pitch detail page (`app/(app)/pitches/[id]/page.tsx`) had 7 side-by-side buttons in the top bar (Preview, Visual Edit, Edit med AI, Stats, Download PDF, Save as Template, Delete). On a narrow laptop the bar ran off the right edge and the red Delete button clipped at the viewport edge. His fix, verbatim: keep **Preview** as a button and collapse everything else into a single **Tools** dropdown.
2. **List had no delete.** The ⋯ overflow menu on the Pitches list only offered "Flyt til mappe". He wanted a **Slet pitch** action there too, mirroring the delete we already built in the detail header.

## Changes
### Detail toolbar (`app/(app)/pitches/[id]/page.tsx`)
- Desktop `sm:` toolbar: `[Preview button]` + `[Tools ▾ dropdown]`.
- Tools dropdown items: Visual Edit, Edit med AI (both HTML-only), Stats, Download PDF, Save as Template, `DropdownMenuSeparator`, then **Slet pitch** (destructive-styled). Reuses the existing `deleteOpen` AlertDialog + handlers — no new delete path.
- Download PDF item uses `onSelect` preventDefault so the inline "Genererer PDF…" state stays visible (the one slow action); others close normally.
- Mobile bottom sheet is unchanged.

### List ⋯ menu (`app/(app)/pitches/page.tsx`)
- Both grid and list ⋯ menus gain a `DropdownMenuSeparator` + destructive **Slet pitch** item, gated `userRole !== 'viewer'`.
- New `pendingDelete` state + `deletePitch()` (DELETE `/api/pitches/[id]`, optimistic row removal, toast, clears any selection of that id).
- One shared custom `AlertDialog` confirmation (never a native `confirm`) — matches the detail-page pattern and the project's no-native-dialogs rule.
