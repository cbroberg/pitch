# F019.8 — Fix app-shell horizontal overflow on narrow laptop widths

## Motivation
On Christian's laptop (effective CSS width below ~1323px) the `/pitches` page overflowed horizontally: the content column was wider than the viewport, so a horizontal scrollbar appeared and the `position: fixed` sidebar stayed put while the pitch-row titles slid *under* it (left edge clipped). His words: "alle ikoner kan ses uden at scrolle langt ud til siden" — a hard, long-standing preference against horizontal scrolling.

At wide widths (>=1440px) the bug was invisible (enough headroom), which is why it slipped past earlier verification done at 1280/1440.

## Root cause
`SidebarInset` (the shared shadcn app-shell content column, `components/ui/sidebar.tsx`) was `flex w-full flex-1 flex-col` **without `min-w-0`**. A `flex-1` item defaults to `min-width: auto` (= its content's min-content width), so the column refused to shrink below ~1003px. In the flex row `[sidebar-gap (~320px) | inset (flex-1)]`, once `viewport < 320 + 1003 ≈ 1323px` the whole row overflowed the viewport → horizontal scroll → content under the fixed sidebar.

Measured via Lens (before fix, list view):
- 1440px: overflowX 0 (clean — masked the bug)
- 1120px: scrollWidth 1323 vs clientWidth 1120 → **203px overflow**, 67 elements past the edge
- 1000px: **320px overflow**

## Fix
Add `min-w-0` to `SidebarInset`'s className. This resets the flex column's `min-width` to 0 so it shrinks to the available space; the content inside (rows with `min-w-0` + `truncate` chains, header with `min-w-0` search container) then reflows to fit. One line, at the shared shell — so every authed page inherits the overflow protection, not just Pitches.

## Verification (Lens, after fix)
- 1120px list, content loaded (folder filter + rows): overflowX **0**, 0 wide elements, all header controls + row actions visible
- 1000px list: overflowX **0**
- 1120px grid: overflowX **0**
- 1440px list: overflowX **0** (no regression)
- Visual: titles truncate gracefully instead of overflowing; content offset correctly after the sidebar

## Non-goals
- Not touching the sidebar collapse behaviour, breakpoints, or mobile Sheet.
- Not redesigning the pitches header/rows — only enabling them to shrink.

## Rollout
Single commit to `components/ui/sidebar.tsx` on main → `fly deploy` (arn). Deterministic CSS (identical Tailwind build dev↔prod); dev-verified at the exact breaking widths. Prod-authed Lens not available (F015.3 mint-secret unarmed) so prod confirmed via health + deterministic-CSS argument.
