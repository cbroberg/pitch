# F020.1 — Trailing-slide dedup (exact-hash detect + perceptual jitter-catch)

Parent: **F020 — PDF export correctness**.

## The bug

`captureDeck` in `lib/pdf.ts` decides "did ArrowRight advance the deck?" and, per step, "is this a new slide?". Both used **exact SHA1**. On the LAST slide, ArrowRight is a no-op, but the re-capture is not byte-identical (sub-pixel/PNG-encoder noise), so exact hashing saw a "new" slide and appended it → one trailing duplicate page (17 for a 16-slide deck).

## First attempt + the regression it caused

The first pass replaced exact hashing with a perceptual signature (32×18 greyscale, mean per-pixel distance) at a `0.02` threshold — for BOTH detection and dedup. That over-reached: **anna-sloth-art's deck has a first transition only ~0.0099 apart** (< 0.02), so the detection check read it as "not a deck" and the export fell through to A4-document pagination — a worse regression than the duplicate. Caught by running the deployed algorithm against anna's real deck on prod (result: flattened document instead of 16 slides).

## Correct scope

- **Detection + real-advance stay EXACT-hash.** Any pixel change = the deck advanced (even a subtle first transition). A perceptual check here misreads decks whose adjacent slides look alike.
- **The perceptual signature is used ONLY to catch the trailing re-capture jitter:** in the loop, a frame that is byte-different from the last kept slide but perceptually within `SLIDE_JITTER = 0.006` is treated as "no advance" (jitter of the same final slide), not appended.
- `SLIDE_JITTER = 0.006` sits between measured jitter (~0.00) and the smallest real adjacent gap (~0.01), so subtle-but-real slides are never merged.
- `captureScroll` reverted to its original exact-hash `Set` dedup — it was never the source of the bug.

## Seal

`lib/pdf.test.ts` (vitest): identical → 0; ~0.004 jitter < SLIDE_JITTER (deduped); ~0.012 gap > SLIDE_JITTER (kept — the anna regression guard); 25%-change far above; empty → 1.

## Runtime proof

Ran the corrected `captureDeck` against anna-sloth-art's real deck on the prod volume: **keyboard-deck detected, 16 slides** (not 17, not a document). The stale 17-page export cache was deleted so the next download regenerates correctly.
