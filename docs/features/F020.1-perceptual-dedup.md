# F020.1 — Perceptual dedup for end-of-deck detection

Parent: **F020 — PDF export correctness** (full root-cause + calibration in the epic plan-doc).

## Change

`lib/pdf.ts`: replace exact SHA1 equality in `captureDeck` and `captureScroll` with a perceptual signature.

- `slideSignature(png)` — `sharp(png).greyscale().resize(32,18).raw()` → 576-byte fingerprint (stable across encoder/sub-pixel noise).
- `signatureDistance(a,b)` — mean absolute per-pixel diff, normalized 0–1.
- `SLIDE_SIMILARITY = 0.02` — below this the deck did not advance (same slide).
- `captureDeck` compares each new screenshot to the last KEPT signature; `< threshold` → no advance (keeps the 2-consecutive-stale tolerance) instead of appending.
- `captureScroll` skips a viewport perceptually identical to the last kept one.
- Removed the dead `hash()` + `crypto` import.

## Seal

`lib/pdf.test.ts` (vitest) asserts the dedup decision: identical → 0, sub-pixel jitter → < threshold, a real (~25%) change → > threshold, empty → 1. Runs in CI.
