# F019.6 — Mobile audit sweep: Access token rows + remaining screens

Parent: **F019 — Mobile-first optimization**.

## Motivation

Christian asked for a full Lens sweep of every screen at his iPhone size. His screenshot of the pitch-detail **Access tab** showed the same overlap class as before: the copy/open/revoke icons sat on top of the email, and the `personal` badge under an action button.

## Findings + fixes

- **Access token rows** (`app/(app)/access/page.tsx` `TokenRow` + the pitch-detail Access tab in `app/(app)/pitches/[id]/page.tsx`): the row packed content + 4–5 action buttons in one `justify-between` flex → badges wrapped under the buttons. Fix: `flex-col gap-3` on mobile, `sm:flex-row sm:items-center sm:justify-between`; actions get `flex-wrap ... sm:ml-2`; the token code + badges get `flex-wrap`.
- **Access tab header** (Access Tokens + Send Invite + New Token): stacks on mobile.
- **templates/generate**: the template select value clipped without ellipsis → `[&>span]:line-clamp-1`.

## Verified clean at iphone-15 (Lens)

dashboard, folders, users, templates, templates/generate, pitches/new, pitch stats, pitch edit, settings, help — no layout overlap. The remaining critic flags are false positives (Radix’s hidden native select, react-dropzone’s hidden input, below-fold content on long pages, the sidebar `sr-only` label) or the pre-existing destructive-red “Revoke” contrast (2.04:1) — a design-token a11y item, not a mobile-layout defect, tracked separately.
