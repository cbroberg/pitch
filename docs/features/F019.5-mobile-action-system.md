# F019.5 — Coherent mobile action system (detail bottom sheet + clean list rows)

> Parent: **F019 — Mobile-first optimization**. Sibling of F019.1–F019.4.

## Motivation

After the first mobile pass (F019.1–F019.4) Christian tested on his iPhone and rejected it: **"Holder ikke. Skal være meget bedre."** Two concrete defects remained, both from cramming desktop toolbars into a phone width instead of designing a mobile pattern:

1. **Pitch-detail header** packed 7 actions (Preview, Visual Edit, Edit med AI, Stats, Download PDF, Gem som skabelon, Slet) in one horizontal row → scrolled off-screen (the "wiggle").
2. **Pitches list rows** overlapped: the `Live` badge sat on top of the views count, `HTML` on top of the open-link icon, and the title collapsed to nothing.
3. (found while fixing) The Pitches **top bar** truncated the search placeholder ("Søg i p…") because search shared a row with the folder filter, and the **grid cards** emitted a nested-`<a>` hydration error (card `Link` wrapping inner edit/preview `Link`s).

## Approach — one mobile pattern, desktop untouched

- **Detail page** (`app/(app)/pitches/[id]/page.tsx`): the inline toolbar becomes `hidden sm:flex`; a `sm:hidden` `⋯` button opens a **bottom `Sheet`** (`components/ui/sheet.tsx`, `side="bottom"`) that lists every action as a full-height (h-12) row with the same role/file-type gating and handlers. Navigation rows are wrapped in `SheetClose` so the sheet closes on tap; Slet is styled destructive.
- **List rows** (`app/(app)/pitches/page.tsx`): on mobile the fileType badge, the right-hand views/date cluster and the inline edit/open buttons are hidden; the title gets a full line; a meta line carries `Live` + views + date; edit/open/move live in the `⋯` menu (`Rediger` / `Åbn i ny fane` are `sm:hidden`, the move submenu stays). The `⋯` trigger is `sm:hidden` for viewers (who have inline actions on desktop) to avoid an empty desktop menu.
- **Top bar**: the search + folder container is `flex-col` on mobile, `sm:flex-row`; both children go `w-full` on mobile, inline on `sm`.
- **Grid cards**: outer `Link` → clickable `div` with `router.push` + keyboard handler (matches the earlier Dashboard fix), removing the nested-anchor hydration error.
- **`?view=list|grid`** deep-link added on mount (consistent with the existing `?folder=` URL state) — makes the view mode shareable and lets Lens verify the list view.

## Non-goals

- No new nav paradigm beyond the existing `Sheet` + `dropdown-menu` primitives.
- Desktop layout is explicitly unchanged (all changes are `sm:`-gated).
- The fixed refresh/update box (PWA has no pull-to-refresh) is tracked separately — not in this story.

## Verification

Lens at `iphone-15` (393×852) on pitch-detail, Pitches list (`?view=list`) and grid (`?view=grid`): 0 high-severity composition findings (the only residual flags are hidden desktop buttons = false positives, below-fold cards, and the sidebar `sr-only` label). Desktop 1280 re-verified: inline toolbars intact. `tsc --noEmit` clean.
