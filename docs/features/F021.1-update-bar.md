# F021.1 — Update banner + version endpoint + menu refresh

Parent: **F021 — PWA in-app update** (motivation + approach in the epic plan-doc).

## Files

- `app/api/version/route.ts` — returns `{ version }` from `.next/BUILD_ID` (module-cached; `dev` fallback), `Cache-Control: no-store`, `dynamic = 'force-dynamic'`.
- `lib/pwa.ts` — `forcePwaUpdate()`: update SW registration → delete all caches → `location.reload()`.
- `components/pwa-update-bar.tsx` — fixed top banner; polls `/api/version` (60s + on focus); shows only when the served build id differs from the loaded one.
- `app/(app)/layout.tsx` — renders `<PwaUpdateBar/>` in the shell.
- `components/nav-user.tsx` — adds an “Opdater app” menu item calling `forcePwaUpdate()`.

## Notes

The service worker never intercepts `/api/`, so the version check always hits the network. The banner renders `null` when no update is pending, so it adds no persistent chrome and cannot overlap page headers.
